Сборка программы

Программа Склад написана с использованием библиотеки Qt компании Trolltech. В качестве сервера базы данных используется СУБД MySQL компании MySQL AB.

Сброка программы возможна на любой платформе, поддерживаемой библиотекой Qt и клиенской библиотекой MySQL. Ниже описан порядок сброки для платформы Windows. Используются следующие компоненты:

  • библиотека Qt (Open Source Edition) версии 4.0.0 или выше,
  • компилятор MinGW версии 3.4.2 или выше,
  • комплект заголовочных файлов и библиотек от сервера MySQL (Community edition) версии 5.0.18 или выше.

Библиотека libmysql.lib поставляется в формате для компилятора от Microsoft и требует преобразования в формат GNU Binutils при помощи утилиты reinp.exe которая входит в состав MinGW Utilites. Для простоты дальнейшей работы рекомендуется заголовочные файлы от MySQL просто скопировать в каталог include компилятора MinGW, а файл с libmysql.a скопировать в каталог lib.

Сброка программы Склад осуществляется стандартным для Qt способом. Из каталога с исходными текстами нужно запустить команду qmake, а затем make.

Настройка сервера базы данных

В качестве инструмента администрирования СУБД MySQL рекомендуется программа SQLyog Community Edition кампании Webyog Softworks. С ее помощью осуществляется создание и настройка базы данных.

Установите сервер СУБД MySQL. Программа Склад использует только MyISAM тип таблиц. Поэтому при инсталляции MySQL при выборе типа базы данных (пункт "Please select the database usage.") можно указать "Non-Transactional Database Only". Рекомендуется в качестве стандартной кодировки выбрать UTF8(пункт "Please select the default character set." нужно выбрать "Best Support For Multilingualism").

Дальнейшая работа с сервером MySQL производится при помощи программы SQLyog. Для того, чтобы создать новую базу данных нужно исполнить скрипт sklad4_init.sql, который находится в каталоге admin программы Склад. Это делается при помощи меню DB -> Restore From SQL Dumb... В окне нужно указать файл sklad4_init.sql и нажать кнопку Execute.

Для доступа к базе данных программа Склад использует пользователя с именем "sklad" и паролем "crkfl". Необходимо создать такого пользователя и дать ему следующие права доступа к базе данных sklad4: SELECT, INSERT, UPDATE, DELETE, CREATE_TMP_TABLES.

Затем нужно создать записи пользователей программы, путем редактирования таблицы users. Подробности см. ниже.

Описание структуры базы данных

Общая структура базы данных показана в base.pdf.

Таблица bom2sklad

Служит для преобразования наименований электронных компонентов при обработке BOM-файла. При вводе комплектации в виде BOM-файла наименования некоторых электронных компонентов могут отличаться от принятых в складе.

Поле Описание
id Первичный ключ
name_bom Наименование, соответствующее BOM-файлу
name_sklad Наименование, соответствующее складу

Таблица izdelie

Содержит название изделий. Связана с таблицей заказов.

Поле Описание
id Первичный ключ
zakaz Заказ, к которому относится изделие.
Внешний ключ на табл. zakaz.
name Название изделия

Таблица kompl

Служит для учета непосредственно электронные компоненты, относящихся к той или иной комплектации (поле sostav).

Поле Описание
id Первичный ключ
sostav Состав, к которому относится комплектация
name Наименование РЭК
nominal Номинал РЭК
type Тип РЭК (id из табл. types)
n Количество РЭК в одной комплектации
need Общее количество необходмиых РЭК в с учетом кол-ва комплектаций
snato Количество снятых РЭК.
status Статус данного РЭК
position Позиция данного РЭК в схеме
color Цвет, который показывает состояние РЭК
